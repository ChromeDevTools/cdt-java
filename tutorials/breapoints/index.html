Intro

The Scripts panel allows you to see all the scripts that are part of the inspected page. History buttons, a drop-down list, and standard controls to pause, resume, and step through code are provided below the panel selection icons. A button to force a pause at exceptions is located at the bottom of the window.



You can set the following types of breakpoints on Scripts panel:
  • Regular JavaScript Breakpoints
  • Breakpoints on DOM Mutation Events
  • Breakpoints on XHR
  • Breakpoints on JavaScript Event Listeners

Regular JavaScript Breakpoints

  1. Open Scripts Panel and select "script.js" from scripts drop-down
  2. Set breakpoint on line 19
  3. Move your mouse over this page
  4. You should stop on the breakpoint



  5. Delete the breakpoint by clicking the blue tag breakpoint indicator
  6. Press play or hit F8 to resume
All the breakpoints you have set appear under Breakpoints in the right-hand sidebar. Clicking on the entry jumps to the highlighted line in the source file. Once you have a breakpoint set, right click on the blue tag breakpoint indicator to set a conditional statement for that breakpoint. Type an expression and the breakpoint will only pause only if the condition is true.

Pause on Next JavaScript Statement

  1. Click the Pause button
  2. Move your mouse over this page
  3. You should stop on mouseover handler



  4. Press play or hit F8 to resume

Pause on Exceptions

  1. Click on "Pause on exceptions" button at the bottom of the window to switch to "Pause on all exceptions" mode
  3. You should stop in raiseAndCatchException function



  4. Press play or hit F8 to resume

Pause on Uncaught Exceptions

  1. Click on "Pause on exceptions" button again to switch to "Pause on uncaught exceptions" mode
  3. You shouldn't stop in raiseAndCatchException function since exception is caught
  5. You should stop in raiseException function



  6. Press play or hit F8 to resume

Breakpoints on DOM Mutation Events

  1. Right click on "Parent Element" below and choose "Inspect Element" from context menu
    Parent Element
  2. Right click on element and choose "Break on Subtree Modifications"
  4. You should stop on "appendChild" function call



  5. Press play or hit F8 to resume

Breakpoints on XHR

  1. Click "+" button on "XHR Breakpoints" pane
  2. Type "data.txt" in text input and hit enter
  4. You should stop on "send" function call



  5. Right-click on newly created breakpoint and click on "Remove Breakpoint" context menu item
  6. Press play or hit F8 to resume
XHR breakpoint with empty URL will match any XHR. You can also create XHR breakpoint from Network Panel:
  1. Go to Network Panel and select XHR resource category
  2. Right-click on "data.txt" resource and click "Set XHR Breakpoint" in context menu
  3. XHR Breakpoint will be added to XHR Breakpoints pane

Breakpoints on JavaScript Event Listeners

  1. Expand "Event Listener Breakpoints" pane on the right side of scripts panel
  2. Click on black triangle near "Mouse" entry inside "Event Listener Breakpoints" pane to expand it
  3. Set a breakpoint by clicking on the checkbox near "mouseout" event entry



  4. Move your mouse across the box below:

  6. Press play or hit F8 to resume
